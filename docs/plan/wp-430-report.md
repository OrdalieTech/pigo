# WP-430 — Markdown, highlighting, and themes

WP-430 ports the pinned upstream Markdown presentation and theme behavior at commit `3da591ab74ab9ab407e72ed882600b2c851fae21`. Markdown is parsed with goldmark and rendered by pi-go, while fenced code is highlighted with chroma and mapped onto the same theme tokens upstream assigns to highlight.js.

## Acceptance evidence

F12 contains 71 cases generated by executing upstream's Markdown component against its test corpus, including focused table-context and constrained-width cases. Together with the Go unit tests, they cover headings, inline styles, links, lists, block quotes, code fences and partial fences, tables, HTML, wrapping, padding, backgrounds, source markers, code-block indentation, and cache invalidation. The Go runner compares every rendered line byte-for-byte, including ANSI state changes and trailing padding.

The embedded `dark` and `light` themes preserve upstream's complete token maps. F12 captures every foreground and background ANSI sequence, resolved CSS color, export color, TypeScript and PowerShell highlighting, unknown-language fallback, strict theme-JSON parsing, unknown-token failure, and a width-72 Markdown sample for both themes. Chroma and highlight.js use different grammars by the settled architecture; the adapter maps shared scopes to upstream theme tokens while the generated cases compare scopes both engines expose consistently.

Theme discovery keeps the first custom name in upstream precedence order: project settings, trusted project auto-discovery, user settings, user auto-discovery, packages, explicit additional paths, then extension-discovered paths. F12 proves project-over-user and extension-first-wins collisions with exact diagnostics; custom themes shadow built-ins without a false collision. The registry and controller expose listing, selection, automatic light/dark resolution, and reload plumbing, while `SettingsManager` separately persists successful choices for later interactive assembly.

The only new direct dependencies are `github.com/yuin/goldmark` and `github.com/alecthomas/chroma/v2`, both explicitly approved for the TUI in ARCHITECTURE section 8. The implementation adds no cgo path or external runtime.

# WP-211 authentication verification

WP-211 ports the pinned Anthropic PKCE flow, the shared credential-resolution core, and TS-pi-compatible `auth.json` persistence. The generated F2 auth fixture exercises TS-written API-key and OAuth entries, command and environment interpolation, unknown OAuth fields, ordered mutation, deletion, the exact callback pages, and a byte-exact Go rewrite. Its migration case also covers JavaScript integer-key ordering, duplicate-provider precedence, invalid-entry skipping, numeric normalization, and the legacy rename; the reciprocal verifier opens the Go-written file through pinned TS pi.

Automated verification covers both localhost callback and manual redirect input, PKCE, OAuth state validation, upstream error wrapping, authorization-code exchange, refresh-token rotation against a fake token server, double-checked concurrent refresh, 0600 file mode, malformed-file protection, and external-edit preservation. A pinned TS process holds `auth.json.lock` while a Go writer proves it cannot enter, and Go heartbeats every five seconds so TS's ten-second synchronous stale window cannot evict a live lock. Startup migration runs before help, model listing, model updates, and normal sessions; `pi login anthropic` and `pi logout` use the same core without requiring the TUI.

The real Claude Pro/Max browser flow could not be completed on 2026-07-18 because this host has no `ANTHROPIC_OAUTH_TOKEN`, no `ANTHROPIC_API_KEY`, and no stored Anthropic OAuth credential. Completing that acceptance check requires a Claude Pro/Max account and interactive browser authorization; run `pi login anthropic`, complete the browser callback or paste its final redirect URL, then make one streamed Anthropic request and confirm TS pi reads the resulting `~/.pi/agent/auth.json`. The implementation tracker remains pending until that credential-dependent check is recorded.

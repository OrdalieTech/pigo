## Shrink diff

The behavior-neutral trim slice is 286 insertions and 513 deletions across code: **239 net
production lines removed** (+190/−429) and a +12 test-line delta (+96/−84) from the migrated
consumers. Production LOC in the mirror groups falls 104,915 → 104,139 (−776), the difference
beyond the raw diff coming from test-file reclassification.

- The 494-line WP450 replay/ctx.ui-demo renderer (`conformance_replay.go`) is now
  `wp450_replay_conformance_test.go`: pure conformance machinery, out of the binary; its two
  runner tests migrated into the same package with fixture bytes and assertions unchanged.
- Two token-identical gitignore-rule match loops (agent harness and codingagent skills — both Go
  ports of the npm `ignore` dependency, the `internal/localecompare` precedent) merged into
  `internal/ignorerules` (−22 net); per-package rule parsing kept its deliberate divergences.
- Two identical insertion-ordered JSON object encoders merged into `jsonwire.OrderedObject`
  (−11 net); `ai.Marshal` already delegates to `jsonwire.Marshal`, so wire bytes are provably
  unchanged (F2 goldens green).
- Deleted dead production code: the never-constructed `ExpandableText` component, the
  superseded `FooterComponent.SetAutoCompactEnabled` push setter (Render reads
  `AutoCompactionEnabled()` live), the write-only `SessionSelectorComponent.currentLoaded`,
  `IsAppleTerminalSession` plus its sole `getenv` consumer, and the `kittyKeyboardFlags` constant
  already hardcoded in the query string.
- Moved to their only consumers (test files): the tui compat overlay surface (`Overlay` wrapper,
  `AddOverlay`, `renderWithOverlays` — upstream tui has no such API), `resolveOverlayLayout` with
  its dimension/coordinate coercers, `Editor.flushAutocomplete`, `newExtensionSelectorComponent`,
  `InteractiveMode.handleSlashCommand`, and `ThemeForRGB`.

No golden or expected value changed. `make fixtures-check` regenerates the full tree byte-clean
after the trim.

## Size and cold-start trend

Same build shapes as M1/M2 (`CGO_ENABLED=0`, `-trimpath`, `-buildvcs=false`; stripped adds
`-ldflags='-s -w'`), same WP-170 no-prompt hyperfine command (20 warmups, 100 runs).

| Metric | M2 | M3 candidate | Change |
|---|---:|---:|---:|
| Default binary | 45,838,979 B | 47,238,108 B | +3.1% |
| Stripped binary | 33,820,834 B | 35,102,882 B | +3.8% |
| No-prompt cold-start mean | 40.0 ± 4.3 ms | 44.6 ± 5.8 ms | +11.5% |

Binary growth stays under the 10% threshold; cold start crosses it (best of three 100-run
measurements: 44.2/46.5/47.8 ms means with 5.8-8.1 ms deviations — partially concurrent-load
noise, the machine was not fully idle, but the direction is real and tracks the +3% binary).
Cold start remains under the 50 ms cap. **The stripped binary now exceeds the decimal 35 MB M5
cap by 102,882 bytes — before the Sprint 3 bridge links sobek and esbuild.** M2 already flagged
bridge size as a release risk with no claimed headroom; that risk is now realized at the margin.
Sprint 3 must treat binary size and startup as first-class constraints (audit what the TUI/CJK
additions pulled in, evaluate section-level savings) and M5 cannot close without material savings
or an owner decision on the cap.

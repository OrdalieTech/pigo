## Shrink diff

The behavior-neutral trim slice is 77 insertions and 378 deletions: **301 net lines removed**.
Production accounts for 36 insertions and 228 deletions (**192 net production lines removed**);
tests account for 41 insertions and 150 deletions (**109 net test lines removed**).

- Nine obsolete or test-only production helpers were deleted. Their remaining tests now exercise
  the current runtime/auth/session paths directly or keep test-only inspection in `_test.go`.
- Two identical RPC message-role/text decoders became one `internal/jsonwire` helper, preserving
  upstream's no-separator text-block concatenation.
- Harness prompt substitution replaced one large generalized matcher with the four sequential
  passes used upstream. This is both smaller and faithful to literal default syntax and recursive
  expansion between passes.
- GitHub Copilot's default model IDs are now decoded lazily at real login instead of during package
  initialization; the nil option remains the default sentinel and a `sync.Once` preserves
  race-safe reuse.

No golden or expected value was changed to accommodate the shrink. Fixture changes in the wider
Sprint 1 diff were generated from new pinned TypeScript cases before their corresponding fixes.

## Dead code, duplication, and abstractions

Production `unused` analysis and the full golangci-lint run report zero issues. Removed helpers
covered the retired skeleton model path, unused message persistence, superseded auth wrappers,
test-only visual truncation/session parsing/message construction, and detached-child counting.

`dupl -t100` falls from eight groups to seven. The actionable cross-package RPC decoder clone is
gone. Remaining groups are protocol or rendering branches whose concrete event/wire types differ;
merging them would add indirection without sharing behavior.

The new `ResourceLoader` is an upstream SDK seam with external implementations and is therefore
allowed by the abstraction rule. SessionRepo/FileSystem, extension operations, provider auth, and
tool operation interfaces likewise mirror upstream or have multiple implementations. The audit
found no new single-implementation speculative interface to inline.

## Size and cold-start investigation

Both binaries use `CGO_ENABLED=0`, `-trimpath`, and `-buildvcs=false`; the stripped artifact also
uses `-ldflags='-s -w'`. Hyperfine uses 20 warmups and 100 runs on the same no-prompt command shape
as M1. The cold-start baseline below is an exact rebuild of M1; the published M1 report measured
4.3 ms on its original run.

| Metric | M1 rebuild | M2 after trim | Change |
|---|---:|---:|---:|
| Default binary | 20,743,679 B | 45,838,979 B | +120.98% |
| Stripped binary | 14,581,922 B | 33,820,834 B | +131.94% |
| No-prompt cold-start mean | 4.0 ± 0.2 ms | 40.0 ± 4.3 ms | +36.0 ms / 900% |

The growth is investigated rather than dismissed. The compiled package graph grew from 257 to 463
packages: 90 AWS SDK, 25 MCP-stack, 19 Anthropic SDK, 15 TUI/Markdown, 13 image, and 19 additional
pi-go packages. The stripped delta is primarily `.text` (+6,947,424 B), `.rodata` (+4,449,376 B),
`.gopclntab` (+4,858,859 B), and `.noptrdata` (+2,851,744 B). Isolated exclusion probes measured
required marginal costs of 3,092,480 B for Bedrock/AWS, 1,646,592 B for Anthropic, 1,679,360 B for
MCP, and a 3,305,472 B TUI floor including the 2,007,296-byte CJK segmentation table. Removing
those imports would remove integrated behavior, not dead weight.

One accidental cost was removable: `ai/providers` constructed the default Copilot OAuth flow and
decoded the 475 KB generated model catalog during every process start. Deferring that lookup until
login reduced provider initialization from 7.3 ms, 770,264 allocated bytes, and 17,246 allocations
to 0.032 ms, 5,488 bytes, and 117 allocations. `pi --version` fell from 23.8 to 14.0 ms and the full
no-prompt path from 48.7 to 40.0 ms.

The current mean remains below 50 ms. The stripped binary is 33,820,834 bytes, leaving 1,179,166
bytes under the decimal 35 MB M5 cap, but `cmd/pi` does not yet link sobek or esbuild. Sprint 3 must
treat bridge size as a release risk and find material savings before M5; this report does not claim
future headroom that does not exist.

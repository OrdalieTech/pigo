The final dead-code pass removed three unused entry points and their state: a superseded CLI input
wrapper, an unearned skill sorter, and unused internal semver string storage. `staticcheck U1000`
is clean; deadcode's remaining 26 reports are 25 exported upstream/SDK entry points plus one TUI
interface method, so deleting them would remove supported surface rather than dead implementation.

The fresh release measurement initially failed at 55.7 ms on an isolated CPU with 25
auto-discovered skills. A syscall trace found 4,118 file-stat calls: canonical-path deduplication
re-resolved every earlier path for each new skill, then the default loader resolved the same
package metadata again. A local seen set and reuse of the already-resolved metadata preserve
first-path-wins and reload freshness while reducing minimal session creation from 48–50 ms to
31–33 ms, 6.48 MB to 5.70 MB, and about 48,940 to 38,730 allocations. The original cold-start
harness now passes above; the controlled before/after is 55.7 → 40.9 ms.

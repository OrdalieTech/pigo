# Extension host API inventory

This is the pre-cutover factual inventory that defined extension-host parity. The embedded Sobek implementation paths below are historical and were deleted by D31 after every listed surface moved to `codingagent/extensions/host/`; the signatures and upstream references remain the checklist. `sync` means the extension receives a value before the JavaScript call returns; `async` means the binding returns or accepts a `Promise`. The difficulty labels are `trivial`, `streamed`, `callback`, and `sync-needs-snapshot`.

The host implementation is assembled in `codingagent/extensions/host/host.mjs` and terminates in `codingagent/extensions/registry.go`; it corresponds to upstream `ExtensionAPI` in `.upstream/packages/coding-agent/src/core/extensions/types.ts:1165-1407` and its runtime construction in `.upstream/packages/coding-agent/src/core/extensions/loader.ts:230-393`. Historical bridge locations in the tables identify the audited source that the host replaced.

### Existing `extension_ui_request` wire contract

The current Go request union has the common `{type:"extension_ui_request",id,method}` envelope and these ordered variants: `select {title,options,timeout?}`, `confirm {title,message,timeout?}`, `input {title,placeholder?,timeout?}`, `editor {title,prefill?}`, `notify {message,notifyType?}`, `setStatus {statusKey,statusText?}`, `setWidget {widgetKey,widgetLines?,widgetPlacement?}`, `setTitle {title}`, and `set_editor_text {text}` (`codingagent/modes/rpc_ui.go:14-116`). This is byte-shape parity with the upstream union (`.upstream/packages/coding-agent/src/modes/rpc/rpc-types.ts:237-273`).

Only `select`, `confirm`, `input`, and `editor` create pending requests. Their response is exactly one `{type:"extension_ui_response",id,value:string}`, `{...,confirmed:boolean}`, or `{...,cancelled:true}`; Go correlates by id and resolves on response, cancellation, or timeout (`codingagent/modes/rpc_types.go:114-120`; `codingagent/modes/rpc_ui.go:138-172,199-267`; `codingagent/modes/rpc.go:201-217`). The other five request methods are one-way. There are no partial response, progress, terminal-input, component-render, or interactive widget messages in this precedent (`codingagent/modes/rpc_ui.go:33-116,250-267`; upstream pairing `.upstream/packages/coding-agent/src/modes/rpc/rpc-mode.ts:105-160,253-269`).

RPC mode binds this adapter into each replacement session and intentionally inherits no-op behavior for the unsupported UI members (`codingagent/modes/rpc.go:148-167`; `codingagent/modes/rpc_ui.go:286-377`). `codingagent/session_rpc.go` is session/runtime control code and contains no `extension_ui_request`/`extension_ui_response` declaration; the positive definitions and dispatch are the three `codingagent/modes/rpc*` locations cited above.

## 7. Sobek shim semantics that disappear in a real Node/Bun host

The bridge strips an optional `node:` prefix and supplies partial built-in modules for `fs`, `fs/promises`, `path`/`path/posix`, `os`, `process`, `buffer`, `url`, `util`, `child_process`, `events`, `readline`, `crypto`, `http`, `https`, `module`, `async_hooks`, `diagnostics_channel`, `tty`, `zlib`, `dns/promises`, `net`, and `perf_hooks` (`codingagent/extensions/jsbridge/shims.go:38-90`). A real Node host supplies those modules, filesystem/process/network behavior, module loading, and child processes directly.

The bridge also installs `process`, `Buffer`, `fetch`, `Headers`, `Request`, `Response`, `console`, `global`, `performance`, `setTimeout`/`clearTimeout`, `setInterval`/`clearInterval`, `atob`, `btoa`, `structuredClone`, `TextDecoder`, `Intl.Segmenter`, `TextEncoder`, `URL`, `Symbol.asyncIterator`, `AbortController`, and `AbortSignal` (`codingagent/extensions/jsbridge/shims.go:93-117,1380,1971,2104-2176,2362-2470`; `codingagent/extensions/jsbridge/node_compat.go:80-167,239-288`; `codingagent/extensions/jsbridge/typebox.go:54-128`; `codingagent/extensions/jsbridge/ui.go:805-892`). Node 22/Bun provides the standard globals; only conversion of Go cancellation into the host's real `AbortSignal` remains an API-bridge concern.

The abort shim specifically supplies `new AbortController().signal`, `controller.abort(reason?)`, `AbortSignal.timeout(ms)`, and `AbortSignal.any(signals)`; its signal-instance surface is the six-row table in section 1 (`codingagent/extensions/jsbridge/ui.go:801-892`; `codingagent/extensions/jsbridge/context.go:668-794`).

Separate from Node compatibility, Sobek currently supplies virtual SDK/package facades: `pi` and coding-agent package aliases, embedded TypeBox root/compile/value, partial `pi-ai`/compat and `pi-tui`, legacy `@mariozechner/*` aliases, plus exported helpers such as `defineTool`, `CustomEditor`, `Key`, `isToolCallEventType`, frontmatter/truncation helpers, and session SDK helpers (`codingagent/extensions/jsbridge/typebox.go:141-185,345-388,502-705`; `codingagent/extensions/jsbridge/sdk_facade.go:20-253`). The real upstream packages expose their SDKs through the coding-agent, AI, and TUI barrels (`.upstream/packages/coding-agent/src/index.ts:1-149,240-300`; `.upstream/packages/ai/src/index.ts:1-46`; `.upstream/packages/tui/src/index.ts:1-114`). These are package-resolution compatibility surfaces, not Node built-ins, so moving to a real runtime removes the Sobek implementations but does not by itself make the named SDK exports available.
